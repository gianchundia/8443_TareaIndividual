Aprendamos Juntos

El lenguaje de marcado extensible (XML) y la notación de objetos de JavaScript (JSON), son los dos lenguajes más comunes para el intercambio de datos en la web.

Autor: Génesis Anchundia

Empecemos

XML

El lenguaje XML no tiene etiquetas predefinidas. Las etiquetas como to y from no están definidas en ningún estándar XML. Estas etiquetas son "inventadas" por el autor del documento XML. HTML funciona con etiquetas predefinidas como p, h1, table, etc. Con XML, el autor debe definir tanto las etiquetas como la estructura del documento.

Muchos sistemas informáticos contienen datos en formatos incompatibles. El intercambio de datos entre sistemas incompatibles (o sistemas actualizados) es una tarea que requiere mucho tiempo para los desarrolladores web. Se deben convertir grandes cantidades de datos y, a menudo, se pierden datos incompatibles.

XML almacena datos en formato de texto sin formato. Esto proporciona una forma independiente de software y hardware de almacenar, transportar y compartir datos. XML también facilita la expansión o actualización a nuevos sistemas operativos, nuevas aplicaciones o nuevos navegadores, sin perder datos. Con XML, los datos pueden estar disponibles para todo tipo de "máquinas de lectura" como personas, computadoras, máquinas de voz, fuentes de noticias, etc.

- XML significa lenguaje de marcado extensible.
- XML es un lenguaje de marcado muy parecido a HTML.
- XML fue diseñado para almacenar y transportar datos.
- XML fue diseñado para ser autodescriptivo.

JSON

JSON es un formato de texto sencillo para el intercambio de datos. Se trata de un subconjunto de la notación literal de objetos de JavaScript, aunque, debido a su amplia adopción como alternativa a XML. JSON es un formato de texto totalmente independiente del lenguaje de programación, pero utiliza términos que son comúnmente conocidos por los programadores, entre ellas:

C++
C#
Java
JavaScript

Semejanzas y diferencias

XML JSON
Es un poco más detallada que JSON, la sintaxis XML se parece a la de las etiquetas HTML. La sintaxis JSON se parece a la del objeto JavaScript.
Es mejor para manejar escenarios más complejos que involucran esquemas e implementar restricciones de datos estrictas. JSON es un lenguaje de intercambio de datos simple, es mejor en el manejo de escenarios más simples relacionados con el intercambio de datos, la interacción de JavaScript y la velocidad.
Admite comentarios y espacios de nombres, e incluso contiene capacidades de visualización. Los archivos JSON pueden contener matrices, más tipos de datos (números, booleanos, cadenas) y un método simple para representar un valor nulo.
Es mejor para la configuración. Es mejor para los servicios web.
Los datos XML se pueden cambiar a otros formatos como texto sin formato. Los datos JSON no se pueden cambiar a otros formatos.
XML está orientado a documentos. El mapeo es bastante fácil, ya que está orientado a datos.
Puede ser usado como un formato de intercambio para que los usuarios muevan datos entre aplicaciones similares. JSON es más inseguro debido a que está basado en JavaScript y no tiene un “parser” de navegador.


SEMEJANZAS
Son formatos populares para el intercambio de datos.
Tanto JSON como XML son "autodescriptivos" (legibles por humanos).
Tanto JSON como XML son jerárquicos (valores dentro de valores).
Muchos lenguajes de programación pueden analizar y utilizar tanto JSON como XML.
Tanto JSON como XML se pueden obtener con XMLHttpRequest.
Tanto JSON como XML se pueden usar para recibir datos de un servidor web.

AJAX

Ajax es una tecnología que permite hacer consultas por HTTP de manera asíncrona contra el servidor.
Bajo esta definición, que puede no ser demasiado clara para algunas personas con poca experiencia en el entorno web,
encontramos una de las herramientas más comunes en sitios, que han permitido evolucionar
las aplicaciones web para que sean tan complejas como las conocemos hoy.
